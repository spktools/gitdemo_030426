Version Control System :

--> LVCS   -- Local Version Control System 

•	Stores versions only on your system 
•	No sharing capability 
•	Risky (if system fails → data lost) 


--> CVCS  -- Centralized Version Control System 

•	One central server stores all code 
•	Example: SVN 
•	Developers connect to server to work 
❌ Problem:
•	If server is down → no work possible 
•	Single point of failure 



--> DVCS  --  Distributed Version Control System -- GIT 
•	Every user has full copy (local + history) 
•	Works offline 
•	Faster and safer
